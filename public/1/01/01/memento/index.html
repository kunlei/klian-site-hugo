Memento Pattern

 Posted on January 1, 0001  |  Kunlei Lian

Memento Pattern is one of the behavioral design patterns defined in the original GoF book. It is useful when we need to save the state of an object and restore it to a saved state at a later time. There are three components in this pattern:

  • Originator: this is the class whose state needs to be saved and restored.
  • Memento: this is the class used to capture the state of the originator.
  • Caretaker: this is the class taking care of the saved mementos.

One example I can think of is the driver seat in my car, which provides the capability to set two positions. Since I am physically bigger, I’d like a position where the seat is far from the steering wheel. My wife, on the other hand, prefers a shorter distance between them. Once set, the seat will adjust to preset position according to the position identifier. Our car installs a switch which allows adjustment of driver seat, a pretty nice feature!

To implement this feature, let’s define a DriverSeat class, which is the originator of the pattern described above.

 1/**
 2 * Originator
 3 * @author Kunlei Lian
 4 */
 5public class DriverSeat {
 6    private String position;
 7
 8    public DriverSeat() {
 9    }
10
11    public void setPosition(String position) {
12        this.position = position;
13    }
14
15    /**
16     * Create position memento based on current position
17     * @return new memento object
18     */
19    public Memento createMemento() {
20        return new Memento(position);
21    }
22
23    /**
24     * Restore to given position saved in memento
25     * @param memento input memento
26     */
27    public void setMemento(Memento memento) {
28        this.position = memento.getPosition();
29    }
30
31    @Override
32    public String toString() {
33        return "DriverSeat{" +
34                "position='" + position + '\'' +
35                '}';
36    }
37}

We also need a class to save a snapshot of the DriverSeat class per user request:

 1/**
 2 * Memento
 3 * @author Kunlei Lian
 4 */
 5public class Memento {
 6    private String position;
 7
 8    public Memento(String position) {
 9        this.position = position;
10    }
11
12    public String getPosition() {
13        return position;
14    }
15}

The class below simulates part of the car’s internal system to save user-defined seat positions, which also includes a demo of the pattern.

 1/**
 2 * Caretaker
 3 * @author Kunlei Lian
 4 */
 5public class Caretaker {
 6    private Map<Integer, Memento> positions;
 7
 8    public Caretaker() {
 9        positions = new HashMap<>();
10    }
11
12    public void savePosition(int posId, Memento memento) {
13        positions.put(posId, memento);
14        System.out.println("DriverSeat position = " + posId + " state = \"" + memento.getPosition() + "\" saved!");
15    }
16
17    public Memento getPosition(int posId) {
18        return positions.get(posId);
19    }
20
21    public static void main(String[] args) {
22        DriverSeat driverSeat = new DriverSeat();
23        Caretaker caretaker = new Caretaker();
24
25        // create first seat memento
26        driverSeat.setPosition("User Position 1");
27        caretaker.savePosition(1, driverSeat.createMemento());
28
29        // create second seat memento
30        driverSeat.setPosition("User Position 2");
31        caretaker.savePosition(2, driverSeat.createMemento());
32
33        // restore to designated user position
34        driverSeat.setMemento(caretaker.getPosition(1));
35        System.out.println(driverSeat.toString());
36
37        driverSeat.setMemento(caretaker.getPosition(2));
38        System.out.println(driverSeat.toString());
39    }
40}

Simulation results:

1DriverSeat position = 1 state = "User Position 1" saved!
2DriverSeat position = 2 state = "User Position 2" saved!
3DriverSeat{position='User Position 1'}
4DriverSeat{position='User Position 2'}

I think the idea of this pattern is to allow the caretaker saving previous states of the originator without knowing its internal representation. Also, the originator may just save part of its state. A good example of this pattern used in JDK can be found in classes implementing the Serializable interface. Object serialization is the process of converting an object’s state to a series of bytes which can be saved to persistent storage and later used to restore the original object.
